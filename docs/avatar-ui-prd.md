# WolfBot 前端 UI 改造 PRD — 头像系统 & 网易狼人杀风格界面

## 1. 项目信息

| 字段 | 值 |
|------|------|
| Language | 中文 |
| Programming Language | Vue3 + TypeScript + Pinia + Element Plus |
| Project Name | wolfbot_avatar_ui |
| 原始需求 | 为 WolfBot AI 狼人杀 Web 游戏增加头像系统，并参考网易狼人杀风格大改前端界面 |

## 2. 产品定义

### 2.1 Product Goals

1. **沉浸感**：通过圆桌布局、头像系统、昼夜氛围切换，让玩家获得桌游级沉浸体验，而非"看后台管理系统"
2. **信息可读性**：玩家状态（存活/死亡/警长/发言中）一目了然，关键操作区不被信息流淹没
3. **人机区分**：真人玩家与 AI 机器人在视觉上清晰可辨，降低混淆成本

### 2.2 User Stories

1. As a **玩家**, I want to **看到所有玩家以圆桌/环形布局围绕排列** so that 我能直观感受到"围坐一桌"的游戏氛围，快速定位每个人的位置
2. As a **玩家**, I want to **每个玩家头像旁清晰显示座位号、名字、存活状态和警长标识** so that 我在发言和投票时无需反复确认谁是谁
3. As a **玩家**, I want to **发言中的玩家头像有高亮动画** so that 我能立刻注意到当前轮到谁发言，不会错过关键信息
4. As a **玩家**, I want to **夜间界面自动切换为暗色调并配合氛围效果** so that 我能感受到昼夜交替的游戏节奏，增强紧张感
5. As a **玩家**, I want to **AI 机器人和真人玩家使用不同风格的头像** so that 我一眼就能区分谁是真人、谁是 AI

## 4. UI 设计要点

### 4.1 整体布局（GamePlay.vue 重构）

**当前布局**：
```
┌──────────────────────┬──────────┐
│                      │          │
│    操作区（左）       │ 信息流   │
│    GameStatus        │ PlayerList│
│    CountdownTimer    │ Announce │
│    RoleCard          │          │
│    ChatBox           │          │
│    VotePanel         │          │
│    NightAction       │          │
│                      │          │
└──────────────────────┴──────────┘
```

**目标布局**（参考网易狼人杀）：
```
┌─────────────────────────────────────┐
│  顶部栏：GameStatus + CountdownTimer│
├─────────────────────────────────────┤
│                                     │
│        ┌─ 圆桌区域（中央） ─┐        │
│       /   玩家头像环形排列    \       │
│      │   (PlayerSeat × N)    │      │
│       \   中央：公告/RoleCard /       │
│        └────────────────────┘        │
│                                     │
├─────────────────────────────────────┤
│  底部操作栏：ChatBox + 操作按钮       │
│  (NightAction / VotePanel 等)       │
└─────────────────────────────────────┘
```

**关键改动**：
- 从两栏 Grid → 上下三段式（顶栏 + 中央圆桌 + 底部操作）
- 圆桌区域占据视觉重心，屏幕高度占比 ≥ 55%
- 底部操作栏固定，不随滚动消失
- 右侧信息流（PlayerList + Announce）融入中央区域，PlayerList 变为环形座位，Announce 变为中央浮动公告

### 4.2 PlayerSeat 组件设计

每个座位卡片结构：

```
┌──────────────┐
│  🏅警长角标   │  ← 左上角，仅 isSheriff 时显示
│              │
│   ╭─────╮   │
│   │ 头像 │   │  ← 圆形 64×64px，死亡时灰度 + 半透明
│   ╰─────╯   │
│   ① 张三     │  ← 座位号 + 名字
│   [存活]     │  ← 状态标签
└──────────────┘
```

- **头像**：64×64 圆形，`border-radius: 50%`，真人默认头像 vs AI 头像
- **座位号**：头像左下角小圆形数字徽章（类似微信未读数）
- **名字**：头像下方居中，12-14px
- **警长角标**：头像右上角金色警徽图标
- **发言高亮**：头像外圈 3px 发光边框 + `box-shadow` 脉冲动画（1.5s 循环）
- **死亡状态**：头像 `filter: grayscale(1) opacity(0.5)` + 右下角红色 ✕ 图标

### 4.3 圆桌排列算法

- 根据玩家数量（6/9/12）动态计算环形排列
- 使用 CSS `transform: rotate() translateX() rotate()` 将 N 个座位均匀分布在椭圆上
- 椭圆横轴 = 容器宽度 × 0.42，纵轴 = 容器高度 × 0.38（适配宽屏）
- 自己的座位固定在正下方（6 点钟位置），其余按座位号顺时针排列

### 4.4 ChatBox 气泡样式

```
他人/AI消息（靠左）：              自己消息（靠右）：
┌─────────────────┐              ┌─────────────────┐
│ [头像]  张三      │              │        我    [头像]│
│         消息内容  │              │  消息内容          │
│         12:30    │              │        12:30      │
└─────────────────┘              └─────────────────┘
```

- 自己消息：主色调气泡，靠右对齐
- 他人消息：灰色气泡，靠左对齐，带头像缩略图
- AI 消息：AI 标识色（如蓝紫色）气泡，靠左对齐，头像为 AI 专用头像
- 系统公告：居中，无气泡，灰色小字

### 4.5 RoleCard 视觉升级

```
┌──────────────────────┐
│  🔮 预言家            │  ← 角色图标 + 角色名
│  ──────────────      │
│  阵营：好人           │  ← 阵营标签（好人绿/狼人红）
│  技能：每晚可查验一人  │  ← 一句话技能描述
│  身份                │
└──────────────────────┘
```

- 每个角色一个专属 SVG 图标（狼/预言家/女巫/守卫/猎人/白痴/平民）
- 阵营用颜色区分：好人=绿色标签，狼人=红色标签
- 游戏未开始时显示"身份未分配"占位

### 4.6 昼夜氛围切换

- **白天**：浅色主题（现有 Element Plus 默认色调）
- **夜间**：暗色主题
  - 背景色：`#1a1a2e` → `#16213e` 渐变
  - 中央圆桌区域：添加半透明月亮/星空 SVG 背景
  - 所有卡片/面板：`background: rgba(255,255,255,0.08)` 磨砂玻璃效果
  - 文字颜色自动切换为浅色
- **切换方式**：通过 CSS 变量 + `data-theme="dark"` 在根元素上切换，而非逐个组件修改
- **关键变量**：`--bg-primary`, `--bg-card`, `--text-primary`, `--text-secondary`, `--accent-color`

### 4.7 头像资源规格

| 类型 | 文件名 | 格式 | 尺寸 | 说明 |
|------|--------|------|------|------|
| 真人默认头像 | `avatar-human.svg` | SVG | 128×128 | 简约人形轮廓，中性色 |
| AI 机器人头像 | `avatar-ai.svg` | SVG | 128×128 | 机器人/芯片风格，与真人明显区分 |
| 角色图标 × 7 | `role-wolf.svg` 等 | SVG | 64×64 | 每个角色一个扁平风格图标 |
| 警长徽章 | `badge-sheriff.svg` | SVG | 24×24 | 金色警徽 |
| 死亡标记 | `badge-dead.svg` | SVG | 24×24 | 红色 ✕ 或骷髅 |

## 5. 技术要点

### 5.1 Player 数据模型变更

```typescript
// types/game.ts — Player 接口扩展
export interface Player {
  id: string;
  name: string;
  seatNumber: number;
  role: RoleType;
  isAI: boolean;
  isAlive: boolean;
  isSheriff?: boolean;
  avatarUrl?: string;  // 新增：自定义头像 URL，为空则使用默认头像
  lastGuardTargetId?: string | null;
  antidoteUsed?: boolean;
  poisonUsed?: boolean;
  isSpectator?: boolean;
}
```

### 5.2 头像计算逻辑（前端）

```typescript
// utils/avatar.ts（新增）
export function getPlayerAvatar(player: Player): string {
  if (player.avatarUrl) return player.avatarUrl;
  return player.isAI ? '/src/assets/avatars/avatar-ai.svg' : '/src/assets/avatars/avatar-human.svg';
}
```

### 5.4 暗色主题实现方案

- 在 `src/styles/` 下新增 `theme-dark.css`，定义暗色 CSS 变量
- `GamePlay.vue` 根元素绑定 `:data-theme="isNight ? 'dark' : 'light'"`
- 所有子组件使用 CSS 变量而非硬编码颜色
- Element Plus 组件通过 `--el-bg-color` 等变量覆盖实现暗色适配
